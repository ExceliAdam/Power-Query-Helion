# 8. Kolumna warunkowa i funkcje logiczne

## Liczba nadgodzin

Formuła generowana przez interfejs (wymagająca poprawy ze względu na typy danych):
```powerquery
= Table.AddColumn(#"Wstawiono różnicę czasu", "Nadgodziny", each if [Czas pracy] > #time(8, 0, 0) then "[Czas pracy] - 8:00" else 0)
```

Poprawiona formuła uwzględniająca typy danych `time` oraz `duration`:
```powerquery
= Table.AddColumn(#"Wstawiono różnicę czasu", "Nadgodziny", each if [Czas pracy] > #time(8, 0, 0) then [Czas pracy] - #time(8, 0, 0) else #duration(0, 0, 0, 0))
```

## Instrukcja warunkowa

Ogólny schemat zapisu funkcji logicznej:
```powerquery
if <test logiczny> then <wynik zwracany, gdy test logiczny jest spełniony> else <wynik zwracany, gdy test logiczny nie jest spełniony>
```

Zagnieżdżanie if:
```powerquery
= Table.AddColumn(#"Zmieniono typ", "Premia", each
if [Sprzedaż] < 10000 then 0 else
  if [Sprzedaż] < 25000 then 500 else
    if [Sprzedaż] < 50000 then 1500 else
      if [Sprzedaż] < 100000 then 3000 else 5000)
```

## Przykłady zapisu czasu i czasu trwania

Zapis godziny 8:00:
```powerquery
#time(8, 0, 0)
```

Zapis zerowego czasu trwania:
```powerquery
#duration(0, 0, 0, 0)
```

## Łączenie operatorów logicznych (and, or, not)

Instrukcja sprawdzająca złożone warunki (dział IT lub dział HR ze stażem min. 10 lat):
```powerquery
if [Dział] = "IT" or ([Dział] = "HR" and [Staż pracy] >= 10) then "TAK" else ""
```

Zastosowanie operatora zaprzeczenia:
```powerquery
not ( [Dział] = "IT" )
```

## Precyzyjne obliczanie stażu pracy

Formuła obliczająca pełną liczbę przepracowanych lat z uwzględnieniem dnia i miesiąca:
```powerquery
Date.Year(DateTime.LocalNow()) - Date.Year([Data zatrudnienia]) -
(if Date.Month(DateTime.LocalNow()) < Date.Month([Data zatrudnienia]) or
  (Date.Month(DateTime.LocalNow()) = Date.Month([Data zatrudnienia]) and
  Date.Day(DateTime.LocalNow()) < Date.Day([Data zatrudnienia]))
then 1
else 0)
```
