# 9. Obsługa błędów

## Zmiana typu danych

Zmiana typów kolumn z przypisaniem typu `any` dla kolumny zawierającej błędy:
```powerquery
= Table.TransformColumnTypes(Źródło,{{"ID Zamówienia", Int64.Type}, {"Cena", Int64.Type}, {"Kod Rabatowy", type text}, {"Rabat", type any}})
```

## Zamiana błędów

Zamiana błędów w kolumnie Rabat na zero:
```powerquery
= Table.ReplaceErrorValues(#"Zmieniono typ", {{"Rabat", 0}})
```

## Warunkowa obsługa wartości null

Formuła korygująca wynik dzielenia w przypadku napotkania wartości pustych:
```powerquery
if [Koszt] / [Godziny] = null then 0 else [Koszt] / [Godziny]
```

## Obsługa nieskończoności

Zamiana dodatniej nieskończoności na konkretną wartość liczbową:
```powerquery
= Table.ReplaceValue(#"Dodano kolumnę niestandardową",#infinity,1,Replacer.ReplaceValue,{"Stawka godzinowa"})
```

Zapis ujemnej nieskończoności w kodzie M:
```powerquery
-#infinity
```

## Konstrukcja try...otherwise

Zastosowanie mechanizmu obsługi błędów do obliczeń (odpowiednik JEŻELI.BŁĄD):
```powerquery
try [Cena Jednostkowa] * [Ilość] otherwise 0
```

Użycie samego słowa kluczowego `try` w celu uzyskania rekordu z informacją o błędzie:
```powerquery
try [Cena Jednostkowa] * [Ilość]
```

## Zapis typów danych w kodzie M

Sposób zapisu złożonych typów danych wewnątrz formuł:
* **Tabela:** `type table`
* **Lista:** `type list`
* **Rekord:** `type record`
